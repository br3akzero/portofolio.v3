OpenCode with Claude Pro/Max

OpenCode is a terminal-based AI coding agent. It supports Claude natively, and it can authenticate using your Claude Pro or Max subscription via OAuth - the same way Claude Code does. No API key needed.

There is a problem though. OpenCode's built-in token refresh no longer works. Once your OAuth access token expires, OpenCode locks you out. You are left staring at an auth error despite having an active paid subscription.

This is an artificial restriction. You are paying for access to the API. The tokens are yours. Anthropic stores them in the macOS Keychain when you log into Claude Code, and they are perfectly valid - OpenCode just fails to refresh them on its own.

The Fix

opencode-inject is a small shell script that bridges the gap. It reads your current Claude Code OAuth credentials directly from the macOS Keychain and writes them into OpenCode's auth.json file. Run it whenever your session expires and OpenCode is back online immediately.

Here is what it does under the hood:

  • Reads the Claude Code-credentials entry from the macOS Keychain using security
  • Extracts the accessToken, refreshToken, and expiresAt fields
  • Writes them into ~/.local/share/opencode/auth.json under the anthropic OAuth key

No third-party dependencies. No network calls. Pure shell and Python 3, both already on your Mac.

Requirements

  • macOS (Keychain access via security CLI)
  • Claude Code installed and authenticated with a Pro or Max subscription
  • OpenCode installed

Quick Start

  1. Auth with Claude Code

    Run claude in your terminal to authenticate before using the inject script.

    Warning: Make sure your Claude Code session does not have stale tokens. If you have not launched Claude Code in a while, open it and verify it connects successfully before running the inject script. Stale or expired tokens in the Keychain will be copied as-is and will not work.
  2. Download the script

    The script is hosted on Codeberg: codeberg.org/breakzero/opencode-inject

    Make it executable and drop it somewhere on your $PATH:

    chmod +x opencode-inject
mv opencode-inject ~/.local/bin/
  3. Run it

    Whenever OpenCode complains about an expired token, just run:

    opencode-inject

    That's it. Your subscription tokens are synced and OpenCode works again.
